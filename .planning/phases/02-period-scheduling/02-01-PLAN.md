# Phase 2: 采集/发送周期调度 — 执行计划

## 目标
重构 Power/Sensor/Comm 任务的定时调度机制，支持 1s~12h 独立可配的采集周期和发送周期。

## 需求
PERIOD-01~05

## 变更清单

### Wave 1: 独立发送周期变量

| # | 文件 | 变更 | 说明 |
|---|------|------|------|
| 1 | `main.h` L195后 | 添加 `extern __IO uint32_t curDevSendPeriod;` | 独立发送周期（秒） |
| 2 | `main.c` L76后 | 添加 `__IO uint32_t curDevSendPeriod = 3600;` | 默认1h，与采集周期相同 |

### Wave 2: powerTask 发送定时逻辑

| # | 文件 | 变更 | 说明 |

## 执行顺序
Wave 1 → Wave 2 → Wave 3 → Wave 4 → 验证 (Wave 5延期)

## 关键设计决策
1. **COMM_CMD_SEND_LATEST** 是轻量级 MQTT publish-only 命令（不读Flash，只发sensorValue）
2. **采样触发时同步更新 lastSendTime**，保证采样→发送流程中不额外触发send latest
3. **发送时总是全 GPRS+MQTT 连接**（非持久连接），功耗由外部周期配置负责
4. **MQTT setsensortime 命令**的 upload_intv 直接作为 curDevSendPeriod 的值
5. **现有 transmissionDensity 逻辑保留**，但 curDevSendPeriod 独立控制发送频率
