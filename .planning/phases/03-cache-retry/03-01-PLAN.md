# Phase 3: 断网缓存与补发 — 执行计划

## 目标
实现断网情况下的数据缓存和重连补发机制。按发送周期缓存原始数据 + 时间戳，联网后按 FIFO 顺序补发，保证数据不丢失。

## 需求
CACHE-01~06

## 设计分析

### 现有缓存基础设施（大部分已存在）
| 机制 | 状态 | 说明 |
|------|------|------|
| `sensorDataSave()` → Flash | ✅ 每次采样后写入 | 512字节/条，含时间戳+传感器值 |
| Flash 环形 FIFO | ✅ `FlashWrite()` 自动回绕 | ~10240 条容量 |
| `unSendDataPacket` / `unSendDataMqtt[]` | ✅ 跟踪未发送记录数 | 从发送指针到写入指针的差值 |
| `curGprsSendAddr` / `curMqttSendAddr[]` | ✅ 跟踪已发送位置 | 持久化到专用Flash区域 |
| GPRS连接失败→不发送→数据留在缓存 | ✅ | `gsmTaskInit()` 失败→无操作 |
| MQTT发送失败→重试（不跳过记录） | ✅ | `transport_sendPacketBuffer` 失败→continue |

### 唯一需要修复的缺陷
**GPRS TIME_ERROR 跳过记录**：当服务器返回 TIME_ERROR 时，现有代码会 `curGprsSendAddr += 512` 并 `unSendDataPacket--`，导致该记录永久丢失。

### 新增功能
**离线检测与退避**：连续 GPRS 连接失败 N 次后，进入缓存模式（跳过 GPRS 上电），定期重试。

## 覆盖矩阵

| 需求 | 实现方式 | 覆盖状态 |
|------|----------|----------|
| CACHE-01: 断网按发送周期缓存 | sensorDataSave() 每次采样后写入Flash；离线时COMM_CMD_SEND_LATEST跳过GPRS但数据已缓存 | ✅ |
| CACHE-02: FIFO存储 | FlashWrite() 环形缓冲区，包头含时间戳 | ✅ |
| CACHE-03: 联网后FIFO补发 | 从 curGprsSendAddr / curMqttSendAddr 起始读取最旧数据 | ✅ |
| CACHE-04: 补发完成恢复正常 | unSendDataPacket==0 时切换回实时 | ✅ |
| CACHE-05: 缓存满丢弃最旧 | FlashWrite() 自动回绕覆盖最旧数据 | ✅ |
| CACHE-06: 1秒周期60包/分钟 | Phase2 1s发送周期 + Phase3 离线不丢包 | ✅ |
